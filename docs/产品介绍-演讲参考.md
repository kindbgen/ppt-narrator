# PPT Narrator（PPT 演讲助手）产品介绍

> 线上投屏讲解参考文档 | 预计演讲时长：25-30 分钟

---

## 一、开场 & 背景介绍（约 5 分钟）

### 1.1 演讲开场

大家好，今天给大家介绍一个我开发的开源工具——**PPT Narrator（PPT 演讲助手）**。这是一个集 AI 生成 PPT 和双屏提词器于一体的桌面应用，解决的是我们日常做技术分享、工作汇报时遇到的几个痛点问题。

### 1.2 做这个工具的动机

我相信在座很多人都有过这样的经历：

**痛点一：PPT 制作过程繁琐**

写一份结构清晰、排版好看的 PPT，从搭框架、找模板、调排版到写文案，经常要花一两个小时甚至更久。即使有 AI 工具的辅助，大部分还是停留在"帮你写文案，你自己去贴"的阶段。

**痛点二：演讲时忘词或节奏失控**

真正站上去讲的时候，要么忘词卡壳，要么某个部分讲太久导致超时，要么眼睛一直瞟屏幕上的备注，跟观众零眼神交流。

**痛点三：投屏分享体验割裂**

用 Zoom / 腾讯会议共享屏幕时，演讲者视图和观众视图没有分离。你看到的备注、计时器，观众也全都看到了——体验很业余。

**痛点四：现有工具不够"一站式"**

市面上要么是 AI 生成 PPT 的工具（只到生成就结束了），要么是提词器 App（只解决读稿问题），中间"从内容到 PPT 再到演讲"这条链路是断的。

**所以 PPT Narrator 的定位就是：打通"内容输入 → AI 生成 PPT+讲稿 → 双屏演讲"全流程的 All-in-One 工具。**

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 技术分享 / 内部培训 | 粘贴文档或 Wiki 页面，AI 自动编排成 PPT + 讲稿 |
| 工作汇报 / 述职 | 需要控制时间、有讲稿参考的正式汇报 |
| 线上录课 / 教程录制 | 录制模式下，一个窗口同时看讲稿和 PPT 预览 |
| 线下投影演讲 | 投影仪放 PPT，电脑屏幕看提词器 |

## 二、产品介绍（约 10 分钟）

### 2.1 产品形态

PPT Narrator 有两种运行形态：

- **Web 版**：浏览器直接打开，打开多个标签页分别作为编辑器、演讲者视图、提词器视图
- **桌面版**：Electron 打包的 Windows / macOS / Linux 原生应用，自动管理多窗口布局

桌面版支持自动识别扩展显示器，将演讲者窗口自动放到第二屏，提词器放在主屏。

│                         PPT Narrator 核心流程                         │
│                                                                      │
│  ┌──────────┐    ┌──────────────┐    ┌──────────┐    ┌────────────┐  │
│  │ ① 内容输入 │ ─→ │ ② AI 生成PPT  │ ─→ │ ③ 编辑微调 │ ─→ │ ④ 双屏演讲  │  │
│  └──────────┘    └──────────────┘    └──────────┘    └────────────┘  │

### 2.3 功能亮点详解

#### 亮点一：多种内容输入方式

不是只能粘贴文本。支持三种输入渠道：
1. **手动输入**：直接粘贴 Markdown 或 HTML 内容
2. **文件导入**：上传本地 `.md` 或 `.html` 文件，一键解析
3. **Wiki 集成**：通过 MCP 协议对接 Docmost Wiki 知识库，直接拉取文档

#### 亮点二：灵活的 AI 后端配置

支持四种 AI 后端，适应不同的网络环境和使用场景：

| 后端 | 适用场景 |
|------|---------|
| LLM Gateway | 团队统一代理，内部部署 |
| Claude（Anthropic） | 高质量生成，需要 API Key |
| OpenAI | 兼容 OpenAI 协议的模型 |
| Ollama | 本地部署，完全离线可用 |

支持配置 API Key、模型名称、温度参数等。桌面版使用操作系统级别的安全存储（macOS Keychain / Windows DPAPI）加密保存 API Key。

#### 亮点三：AI 生成 10 种页面布局

这可能是本项目的亮点。AI 不仅仅是生成"标题 + 要点"的单调列表，而是根据内容语义生成 10 种不同的页面布局：

1. **封面页** — 大标题 + 副标题 + 演讲者信息
2. **目录页** — 结构化章节导航
3. **分隔页** — 章节切换过渡
4. **正文页** — 标题 + 内容的常规布局
5. **数据卡片页** — 多卡片并列展示数据/指标
6. **双栏对比页** — 左右对比（方案 A vs 方案 B）
7. **表格页** — 结构化数据展示
8. **时间线页** — 日程/里程碑/演进路线
9. **强调页** — 大段引述/核心观点突出展示
10. **结束页** — 致谢/联系方式

渲染引擎是 Reveal.js，支持自定义 CSS 主题（商务、科技、极简、教育四种视觉风格）。

#### 亮点四：不只是 PPT — 附带完整讲稿

AI 不但生成 PPT，还会为每一页生成：
- **讲述文案（Narration）**：这一页你应该说什么
- **关键词（Keywords）**：快速记忆要点
- **演讲技巧（Tips）**：这一页的演讲建议

这个讲稿就是提词器模式的核心数据来源。

#### 亮点五：真正的双屏演讲体验

这是和竞品拉开差距的地方：

```
真实演讲场景：
┌─────────────────────┐   ┌─────────────────────┐
│   🖥 扩展显示器/投影仪  │   │    💻 你的笔记本屏幕    │
│                     │   │                     │
│   Presenter 窗口    │   │   Narrator 窗口     │
│   全屏 PPT 播放     │   │   ┌───────┬───────┐ │
│   观众看到的内容     │   │   │ 讲稿  │ PPT   │ │
│                     │   │   │ 计时器 │ 预览  │ │
│   你共享这个屏幕     │   │   │ 关键词 │       │ │
│                     │   │   │ 技巧  │       │ │
└─────────────────────┘   │   └───────┴───────┘ │
                          └─────────────────────┘
```

**Narrator（提词器）窗口包含：**
- 左侧（35%）：当前页面的 AI 讲稿、关键词、演讲技巧
- 右侧（65%）：当前页面的 PPT 小窗预览
- 顶部：每页倒计时器（绿/黄/红三级警告）、累计耗时、进度条
- 底部：页码标签快速跳转

**时间管理：** 每页设定推荐时长，倒计时器用颜色提示节奏：
- 🟢 绿色：剩余 > 30 秒 — 节奏正常
- 🟡 黄色：剩余 10-30 秒 — 加快一点
- 🔴 红色：剩余 < 10 秒 — 尽快收尾

**双向同步：** 按键盘 ← → 翻页，Presenter 和 Narrator 窗口实时同步。你在提词器翻页，观众看到的 PPT 也跟着翻。用的是 BroadcastChannel（浏览器）和 Electron IPC（桌面版）实现跨窗口通信，同一个代码库同时支持两种环境。

#### 亮点六：单屏录制模式

如果你只有一个屏幕、要录制视频或者开线上会议不方便做双屏分离，可以用**录制模式**。

录制模式下，Narrator 窗口变成一个左右分栏的单一窗口（35% 讲稿 + 65% PPT 预览），你可以把这个窗口直接共享给会议软件，或者用它来录屏，同时自己看到讲稿内容。

#### 亮点七：安全与离线

- **桌面版安全设计**：`contextIsolation: true`，`nodeIntegration: false`，`sandbox: true`。渲染进程完全沙箱化。
- **API Key 加密存储**：用 Electron `safeStorage` API，在 macOS 上存在 Keychain，Windows 上存在 DPAPI。
- **离线运行**：如果你用 Ollama 本地模型，整个工作流完全不需要联网——内容输入、AI 生成、编辑、双屏演讲，全都在本地完成。

#### 亮点八：演讲项目管理

不是一次性的工具——你的所有演讲项目都存在本地 SQLite 数据库里（通过 sql.js WebAssembly + IndexedDB 持久化），支持：
- 多项目并行生成（每个项目独立进度）
- 项目搜索和查看
- 生成断点续传（中断后从断点继续）
- 快捷演示数据（不用配置 AI，一键生成示例 PPT 体验完整流程）

## 三、实操演示（约 10 分钟）

> 以下是投屏演示的操作流程，打开 Web 版（http://localhost:5173）或桌面版均可。

### 3.1 步骤一：展示首页

**操作：** 打开应用，定位到首页（Home）。

**讲解要点：**
- 这就是内容输入界面。左侧是文本输入区，可以粘贴 Markdown 或 HTML。
- 顶部工具栏：可以导入文件、连接 Wiki。
- 或者点「快速测试」按钮，一键加载示例数据，无需配置任何 AI。

### 3.2 步骤二：配置 AI

**操作：** 点击设置（侧边栏齿轮图标）。

**讲解要点：**
- 四种 AI 后端可选：LLM Gateway、Claude、OpenAI、Ollama
- 填入 API Key（桌面版会加密存储到系统钥匙串）
- 选择视觉风格：商务 / 科技 / 极简 / 教育
- 点「测试连接」验证配置是否正确

### 3.3 步骤三：AI 生成 PPT

**操作：** 回到首页，粘贴或加载内容，选择风格，点击「生成 PPT」。

**讲解要点：**
- AI 进入两阶段生成：第一步分析内容结构生成 PPT 骨架，第二步为每页撰写讲稿
- 进度条实时显示进展
- 生成完自动跳转到编辑器

### 3.4 步骤四：编辑和微调

**操作：** 在编辑器中查看和调整。

**讲解要点：**
- 左侧幻灯片列表，可以拖拽排序
- 点选某一页，右侧显示标题、HTML 内容、时长、讲稿
- HTML 内容支持「可视化预览」和「源码编辑」两种模式切换
- 展开元数据面板：填写演讲者姓名、部门、活动类型、日期、主题风格
- 导出功能：PPT 可导出 PDF / HTML / Markdown；讲稿可导出 PDF / HTML / Markdown

### 3.5 步骤五：开始演讲

**这是最关键的演示环节，需要两个窗口。**

**操作：**
1. 点击编辑器顶部的「开始放映」按钮
2. Presenter 窗口自动打开（桌面版会自动放到扩展屏）
3. Narrator 窗口自动打开（桌面版居中在主屏）

**讲解要点（Presenter 窗口）：**
- 全屏 PPT 播放，右键菜单可控制翻页、全屏切换、结束放映
- 键盘 ← → 翻页，Esc 结束放映
- 这个窗口就是共享给观众的窗口

**讲解要点（Narrator 窗口）：**
- 左侧讲稿区：AI 写好的讲稿、关键词、演讲技巧
- 右侧 PPT 预览：当前页的小窗展示
- 顶部倒计时器：每页时长控制，绿/黄/红颜色警告
- 底部页码标签：快速跳转到任意页
- 键盘操作：← → 翻页，空格暂停/继续计时
- 翻页时两个窗口实时同步

### 3.6 步骤六：展示录制模式

**操作：** 关闭 Presenter 窗口，切换到录制模式。

**讲解要点：**
- 录制模式是单个窗口，左右分栏：35% 讲稿 + 65% PPT 预览
- 适合单屏录课、或者在线会议时直接共享这个窗口
- 演讲者能看到讲稿和计时，观众看到的是干净的 PPT

### 3.7 步骤七：展示项目管理

**操作：** 回到首页，点击「所有项目」。

**讲解要点：**
- 所有演讲项目都在这里，支持搜索
- 演讲数据存在本地 SQLite 数据库里
- 可以打开历史项目重新编辑或演讲

## 四、总结（约 2 分钟）

### 4.1 一句话总结

**PPT Narrator = AI 生成 PPT + 双屏提词器 = 从内容到演讲的完整闭环**

### 4.2 核心优势回顾

| 维度 | 亮点 |
|------|------|
| 内容到演讲 | 全流程覆盖，不是只做到"导出PPT"就结束 |
| AI 生成质量 | 10种布局、4种风格、附带讲稿和演讲技巧 |
| 双屏体验 | 真正的演讲者/观众视图分离，含计时和提词 |
| 安全离线 | 沙箱化桌面应用、Keychain 加密、支持本地模型脱离互联网 |
| 开源免费 | 代码开源，可自行部署和定制 |

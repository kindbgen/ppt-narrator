# PPT Narrator 产品介绍（精简版）

> 线上投屏讲解参考 | 预计时长：15-18 分钟

---

## 一、开场 & 背景（约 4 分钟）

大家好，今天给大家介绍一个我开发的开源工具——**PPT Narrator（PPT 演讲助手）**，一个集 AI 生成 PPT 和双屏提词器于一体的桌面应用。

### 1.1 核心动机：从 Wiki 长文档到精炼 PPT

先聊聊为什么要做这个工具。

我们公司 Wiki 知识库沉淀了大量产品研发文档——技术方案、项目复盘、架构设计等等。这些文档写得很详细，内容价值很高，但篇幅太长，一篇就是几千字。

当你想把某篇文档的内容分享给其他团队、或者向领导汇报的时候，不可能让对方去翻几十页 Wiki。你需要把文档里的精华提炼出来，做成一份十几页的 PPT，让别人十分钟内就能看懂你的核心内容。

问题是：读文档、划重点、搭大纲、排版、写讲稿——整个过程手工做下来，一两个小时起步。而且每个人做 PPT 的风格不统一，有的排版精致，有的只能说是"字都放上去了"，质量参差不齐。

**所以这个工具最核心的出发点就是：打通 Wiki 知识库，通过 MCP 协议直接拉取文档，让 AI 帮你读懂、提炼、排版，生成一份结构清晰、风格统一的 PPT，同时连每一页的讲稿也帮你写好。**

### 1.2 其他几个痛点

除了做 PPT 耗时之外，日常演讲还有几个挺头疼的问题：

- **做 PPT 太慢**：从搭框架、找模板到排版写文案，经常一两个小时过去。即使有 AI 辅助，大多也只是帮你写文案，排版还得自己来。
- **演讲容易忘词或超时**：讲台上记不住要点，或者某个章节讲太投入，后面时间不够用。
- **投屏视图不分离**：用会议软件共享屏幕时，你看的备注、计时器，观众也全看到了，体验很业余。
- **工具链路断裂**：市面上 AI 生成 PPT 的工具不少，提词器应用也有，但从内容到 PPT 再到演讲，中间这条路没人打通。

**所以 PPT Narrator 的定位就是：打通「内容输入 → AI 生成 PPT+讲稿 → 双屏演讲」全流程。**

## 二、产品演示（约 10 分钟）

下面直接打开工具，边操作边讲。

### 2.1 内容输入

首页就是内容输入区。支持三种方式：

- **手动粘贴**：直接贴 Markdown 或 HTML
- **文件导入**：拖拽 `.md` / `.html` 文件
- **Wiki MCP 导入（重点）**：通过 Docmost MCP 协议直接读取公司 Wiki，选择文档一键导入

> 现场演示：打开一篇 Wiki 产品文档，点击导入，几秒后完整内容进入编辑器。

### 2.2 AI 配置

点击设置，四种 AI 后端可选——LLM Gateway、Claude、OpenAI、Ollama。桌面版 API Key 通过系统钥匙串加密存储。另有商务、科技、极简、教育四种视觉风格。

### 2.3 AI 生成

内容导入后选择风格，点击生成，AI 分两步工作：
1. 分析文档结构，生成 PPT 骨架——不是千篇一律的「标题+列表」，而是根据内容语义匹配 10 种布局：封面、目录、分隔页、正文、数据卡片、双栏对比、表格、时间线、强调页、结束页
2. 为每一页撰写讲稿、关键词和演讲技巧

> 现场演示：选一篇文档 → 选风格 → 点击生成 → 等待完成进入编辑器。

### 2.4 编辑与导出

编辑器左侧是幻灯片列表，支持拖拽排序。选中某一页，右侧编辑标题、HTML 内容、时长和讲稿。内容编辑支持可视化预览和源码模式切换。可填写演讲者信息、部门、日期等元数据。支持 PPT / 讲稿导出为 PDF、HTML、Markdown。

> 现场演示：快速翻看生成结果，调整某一页内容。

### 2.5 双屏演讲（重点环节）

点击「开始放映」，打开两个窗口：

**Presenter 窗口**——全屏 PPT 播放，这是观众看到的内容。桌面版会自动放到扩展屏或投影仪上，你共享这个窗口即可。← → 翻页，Esc 退出。

**Narrator 窗口**——提词器，只在你主屏显示，观众看不到：

```
┌──────────────────────────────┬──────────────────┐
│  ⏱ 倒计时 01:23 / 02:00 🟢   │   PPT 实时预览     │
│                              │                  │
│  📝 AI 讲稿：                │   ┌──────────┐   │
│  这一页的核心论点是...       │   │          │   │
│                              │   │  Slide   │   │
│  🔑 关键词：提炼、结构化     │   │  预览    │   │
│                              │   │          │   │
│  💡 技巧：可举例说明         │   └──────────┘   │
├──────────────────────────────┴──────────────────┤
│  [1] [2] [3] [4] [5] ...          已用: 05:32  │
└────────────────────────────────────────────────┘
```

核心设计：
- **每页倒计时**：🟢绿色 (>30s) / 🟡黄色 (10-30s) / 🔴红色 (<10s)，颜色自动提醒节奏
- **双向同步**：← → 翻页，Presenter 和 Narrator 实时联动
- **页码标签**：底部可快速跳转到任意页
- **录制模式**：单屏时合并为左右分栏窗口（35% 讲稿 + 65% PPT 预览），适合录课或线上会议

> 现场演示：点击开始放映 → Presenter 在扩展屏、Narrator 在主屏 → 翻页展示同步效果 → 演示计时器变化。

## 三、总结（约 2 分钟）

### 3.1 一句话

**PPT Narrator = AI 提炼 Wiki 长文档 → 生成 PPT + 讲稿 → 双屏提词演讲，全流程闭环。**

### 3.2 核心优势

| 维度 | 亮点 |
|------|------|
| Wiki 打通 | MCP 协议直连公司知识库，一键导入 |
| AI 生成质量 | 10 种布局、4 种风格、附带讲稿和演讲技巧 |
| 双屏体验 | 演讲者/观众视图真正分离，含计时与提词 |
| 安全离线 | 沙箱化桌面应用、Keychain 加密、支持 Ollama 本地模型 |
